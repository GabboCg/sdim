## Package Overview

`sdim` is an R package implementing four factor extraction methods from two related papers:

- **sPCA** — Huang, Jiang, Li, Tong, Zhou (2022), "Scaled PCA: A New Approach to Dimension Reduction", *Management Science*, doi:10.1287/mnsc.2021.4020. Univariate target: each predictor is scaled by its OLS slope on the target before PCA.
- **PCA, PLS, RRA** — He, Huang, Li, Zhou (2023), "Shrinking Factor Dimension: A Reduced-Rank Approach", *Management Science* 69(9). RRA is the main contribution; PCA and PLS are benchmarks. Multivariate target (T×N returns matrix).

Both papers are published in *Management Science*.

Intended for submission to the Journal of Statistical Software (JSS).

## Architecture

Shared helpers live in `R/utils.R` (all dot-prefixed, not exported):
- `.validate_inputs(target, X, nfac)` — dimension checks, coercion; returns `list(target, X, nfac)`
- `.as_numeric_matrix(x)` — validates and coerces `X` to a numeric matrix
- `.standardize_matrix(y)` — centers and scales columns to mean 0, SD 1
- `.winsor(x, p)` — winsorizes a vector at given percentile bounds (0–100 scale)
- `.pc_T(y, nfac)` — extracts `nfac` principal components via SVD on `Y %*% t(Y)`
- `.mat_neghalf(Q)` — computes Q^(-1/2) via eigendecomposition (base R only)

Estimator logic lives in separate files (`R/pca_est.R`, `R/pls_est.R`, `R/spca_est.R`, `R/rra_est.R`). `R/sdim.R` (`sdim()` wrapper + `compare()` generic) is planned but not yet implemented. All estimators return a single S3 class `sdim_fit`. S3 methods for `sdim_fit` live in `R/sdim_fit.R`.

**`sdim_fit` fields (common):** `method`, `factors` (T×K), `lambda` (L×K), `residuals` (T×L, G-space), `eigvals`, `ve2`, `call`. Method-specific optional fields: `beta`, `beta_scaled`, `Xs`, `scaleXs` (sPCA); `pls_weights` (PLS); `gamma` (PCA); `gmm_stat` (RRA).

**`sdim_list`** is a named list of `sdim_fit` objects (one per method). `print.sdim_list` in `R/sdim_fit.R` prints a comparison table. The constructor (`sdim()`) is not yet implemented.

**`eval_factors(ret, factors)`** in `R/eval_factors.R` — exported utility that evaluates extracted factors against a target return matrix. Returns a named vector: `RMSPE` (root-mean-squared pricing error, %), `TotalR2` (total adj-R², %), `SR` (mean |alpha/sigma|), `A2R` (mean |alpha/mean_ret|), as defined in He et al. (2023) §2.4 Equations 19–20.

## Bundled Datasets

Seven `he2023_*` datasets ship with the package (documented in `R/data.R`):
- `he2023_factors` — 70 factor proxies (5 FF + 65 anomaly long-short), 516 months ending 2016-12-01. Used as `X`.
- `he2023_ff48vw`, `he2023_ff30vw`, `he2023_ff17vw`, `he2023_ff48ew` — Fama-French industry portfolios (528 months ending 2017-12-01). Used as `target`.
- `he2023_dacheng202` — 202 Dacheng Xiu portfolios (552 months). Used as `target`.
- `he2023_ff5` — FF5 + momentum + liquidity + RF (652 months). Used for constructing excess returns.

**Date alignment caveat:** `he2023_factors` ends 12 months earlier than the portfolio datasets. Filter dates before passing to estimators.

## JSS-Specific Requirements

The package is targeting Journal of Statistical Software. Key implications:
- `DESCRIPTION` still has placeholder metadata — Title, Authors, Description, and License must be filled in before submission
- `NAMESPACE` uses `exportPattern("^[[:alpha:]]+")` — switch to explicit `@export` tags generated by `devtools::document()` before submission
- JSS requires a vignette (typically a `.Rnw` or `.Rmd`) that serves as the paper; this does not yet exist
- JSS requires all documented functions to have `@examples` that pass `R CMD check --run-donttest`
- Only base R and `stats` are used as dependencies — keep it that way unless there is a strong reason to add dependencies
